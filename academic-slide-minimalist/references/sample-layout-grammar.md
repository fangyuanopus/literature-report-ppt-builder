# Sample Layout Grammar

Use this reference to translate the user's sample PPT style into repeatable layout patterns. The sample deck provides visual grammar, not fixed scientific content or fixed navigation names.

## Core Rule

Choose a layout type before designing each page. A slide should not be an arbitrary collage.

## Layout Types

### 1. Cover

Use for title, translated title, paper citation, presenter, and date. Keep clean and do not add scientific visuals unless the paper's own cover/TOC graphic is appropriate and real.

### 2. Paper Information

Use for title, authors, journal, year, DOI, research object, and 3-5 keywords. A small paper screenshot or citation block is acceptable.

### 3. Contribution Overview

Use for the core problem, strategy, evidence, and result in 3-4 concise blocks. Avoid decorative icons; use text blocks and simple separators.

### 4. Background Gap

Use to explain field need, existing methods, and unresolved contradiction. Prefer real paper background figures; otherwise use text-only logic blocks.

### 5. Route / Logic

Use for author strategy, experimental route, or paper roadmap. Use simple text arrows or boxes only. Do not create fake mechanisms or scientific cartoons.

### 6. Large Figure Evidence

Use when one figure is the main proof. Make the real figure dominant and annotate only the key region, trend, or number.

### 7. Left Text + Right Figure

Use when a brief explanation frames one main real figure. Left side: 2-3 points. Right side: source figure.

### 8. Two-Figure Comparison

Use when two real figures directly compare samples, conditions, before/after states, or source/evidence relationships.

### 9. Three-Column Evidence

Use only for true three-way comparisons. Each column must have a real source figure or a concise evidence block.

### 10. Table Key Rows

Use for literature comparisons or parameters. In the mainline, crop or reconstruct only key rows from the real table image when readable; put full tables in backup.

### 11. Summary / Inspiration

Use for 3-4 conclusions, evidence links, limitations, and what the user can learn from the paper.

### 12. Backup Full Evidence

Use for full SI tables, full fitting parameters, complete method details, and teacher-question support. Keep the same style but mark Backup clearly.

### 13. Closing

Use for “汇报完毕 / 敬请批评指正”. Keep minimal.
