# Speaker Notes Template

Use this reference when the user wants a presentation that can be spoken clearly, or when maximum-quality mode is requested.

## Purpose

A literature-report PPT should not only look good. It should be easy to present. Speaker notes convert the slide logic into a spoken explanation.

## Per-Slide Speaker Note Template

```text
slide number:
section:
spoken purpose:
30-60 second script:
transition from previous slide:
transition to next slide:
likely question:
safe answer:
```

## Writing Rules

- Use natural Mainland Chinese academic oral style.
- Do not read slide text word for word.
- Explain why the figure matters, not only what it contains.
- Keep uncertainty calibrated: distinguish proven results from possible explanations.
- Mention SI only when it helps the audience trust a claim.

## Example Style

```text
这一页主要说明作者为什么要设计这种结构导向剂。前面我们已经看到，已有材料的问题不只是孔径不够大，还包括孔口形状和骨架稳定性。这里作者的思路是通过更大体积、空间匹配度更高的模板剂来支撑超大孔道。需要注意的是，这一页还只是设计逻辑，真正的结构证据会在后面的 STEM 和结构精修结果中展开。
```

## Likely Question Types

Prepare safe answers for:

- Why did the authors choose this method/material?
- Is this evidence direct or indirect?
- Does the SI support the main conclusion?
- Are there limitations in the comparison?
- Can this result be generalized?
- What is the biggest inspiration for our own work?

## Expanded Question Preparation

For maximum-quality runs, use `question-prep.md` to create richer question coverage. Speaker notes may include only the most likely question for each slide, while `possible_questions.md` can include the full list.

## Terminology Consistency

Speaker notes must use the same Chinese terms as the slides and terminology table. Do not introduce a second translation for a key term during oral explanation.

## Backup Slide Reference

When a likely question can be answered by a backup slide, include:

```text
backup reference: Backup slide __ / Fig. S__ / Table S__
```

## Reverse Slide Check

Use `speaker-notes-to-slide-check.md` as a reverse quality test. If a clear 30-60 second explanation cannot be written for a key slide, fix the slide before final delivery.
