# Sample Style Specification

Use this file with `sample-deck-rhythm.md` and the bundled `assets/sample-literature-report.pptx`.

## Role of the Sample Deck

The sample deck is a structure and style reference. It is not a source of scientific content for new papers.

## Structural Style

Default section order:

```text
基本信息 -> 研究背景 -> 研究思路 -> 研究结果 -> 总结启发
```

Most content pages should include:

- a top navigation strip;
- a clear active section marker;
- a large conclusion-style title;
- one main figure area or evidence block;
- short supporting text;
- page number.

## Visual Style

Use:

- white or very light gray background;
- deep academic red for active navigation and key emphasis;
- black for main text;
- gray for inactive navigation and secondary notes;
- thin lines and simple boxes;
- restrained whitespace.

Avoid:

- commercial poster style;
- decorative icons;
- gradients, glows, 3D effects;
- AI-generated schematic decorations;
- excessive card dashboards.

## Text Rhythm

The slide language should sound like a Chinese graduate student giving a group-meeting literature report.

Prefer:

```text
本工作报道了……
该结果表明……
进一步说明……
这说明作者的核心设计在于……
```

Avoid:

```text
赋能
平台化价值
广阔应用前景
结构-性能协同赋能
具有划时代意义
```

## Layout Rhythm

Prefer layouts in this order:

1. one dominant real figure with two or three short notes;
2. two real figures in source-evidence relation;
3. left logic explanation + right main figure;
4. three-column comparison only when the comparison is real;
5. text-only logic page only when no real figure is needed.

## Page Density

Do not copy the sample deck's page count too rigidly. The user's updated requirement is at least 20 pages and unlimited page count if needed. Use more pages to explain logic clearly rather than crowding one slide.


## Adaptive Navigation Style

The visual look of the navigation bar should follow the sample deck, but the words in the navigation bar should be derived from the current paper. Do not force every paper into the same `基本信息 | 研究背景 | 研究思路 | 研究结果 | 总结启发` labels.

When a paper has a long result section, split the navigation into audience-friendly evidence layers such as `性能验证`, `结构证据`, `机理解释`, `应用验证`, or domain-specific equivalents. Keep labels short and consistent across all pages.

## Layout Grammar Link

Use `sample-layout-grammar.md` to choose a stable page type before layout. Prefer repeatable page grammars over ad hoc generated layouts. This keeps multiple papers and multiple image batches closer to the user's accepted sample style.

Use `figure-readability-standard.md` before accepting any figure page and `annotation-language.md` before adding labels or callouts.
