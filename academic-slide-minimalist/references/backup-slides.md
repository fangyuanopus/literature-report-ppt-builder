# Backup Slides

Use this reference when creating a complete literature-report deck that may be questioned by a teacher, advisor, or seminar audience.

## Purpose

Backup slides are optional support pages placed after the closing page or delivered as a separate backup section. They should not interrupt the main story, but they should help answer likely questions.

## When to Create Backup Slides

Create or propose backup slides when:

- SI contains important but dense evidence;
- methods, synthesis conditions, or controls are likely to be questioned;
- a result slide relies on a simplified crop of a larger figure;
- the paper has mechanism claims that require additional support;
- performance comparisons require detailed conditions;
- stability/recycling data exist but are too detailed for the main deck.

## Suggested Backup Topics

- detailed synthesis or preparation conditions;
- original full figure panels before cropping;
- extra PXRD/XRD, NMR, XPS, FTIR, Raman, TG/DSC, BET, SEM/TEM/STEM evidence;
- structure refinement details;
- control experiments;
- additional substrates or sample comparisons;
- repeated experiments, stability, recycling, long-time tests;
- calculation details or parameter tables;
- author limitations and unresolved questions.

## Backup Slide Rules

- Backup slides still must use real paper/SI figures only.
- Use simpler titles such as “补充：……”.
- Keep source labels visible: `Fig. Sx`, `Table Sx`, or source page.
- Do not move weak evidence into the main deck just to reach 20 pages.
- Do not hide major uncertainty only in backup; if it changes the main conclusion, mention it in the main deck.

## Backup Slide Naming

Use separate numbering or labels:

```text
Backup 1: 补充：合成条件与样品命名
Backup 2: 补充：完整 PXRD 对照
Backup 3: 补充：循环稳定性数据
```

## Delivery

In maximum-quality mode, include backup slides when useful. They may be:

- appended after “汇报完毕”; or
- returned as `backup_slides.pptx`; or
- described in `backup_slide_plan.md` if not generated.


## Mainline Boundary

Use `mainline-backup-boundary.md` before deciding whether a dense SI page belongs in the main story or backup. Backup slides should support questions and preserve details; they should not interrupt the main narrative unless the evidence is necessary for the next mainline claim.

Backup slides must still follow the same adaptive navigation system, with `Backup` as the active state.
