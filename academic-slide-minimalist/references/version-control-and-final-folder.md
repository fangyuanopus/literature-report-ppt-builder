# Version Control and Final Folder

Use this reference to prevent old or mismatched page images from entering the final PPT.

## Core Rule

Only assemble the final PPT from one approved final image folder.

## Folder Convention

Use clear versioned folders:

```text
image2_pages_v1/
image2_pages_v2/
image2_pages_redraw_batch_01/
image2_pages_unified_final/
```

The final PPT must use:

```text
image2_pages_unified_final/
```

## Final Filename Convention

Inside the final folder:

```text
slide_01.png
slide_02.png
slide_03.png
...
```

Filenames must match `deck_order_map.md` and visible slide numbers whenever possible.

## Old-Page Rule

Do not mix old images into the final folder unless they have been inspected and renamed into final slide order. If an old page is reused, record it in `image_generation_status.md`.

## Final Folder Checklist

Before assembly, verify:

- no duplicate slide files;
- no missing slide files;
- no non-final experimental images;
- image dimensions are consistent;
- page numbers and filenames match;
- navigation style is consistent.
