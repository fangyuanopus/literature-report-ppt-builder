# Real Figure Annotation Rules

Use this reference before creating, editing, or annotating any scientific slide image.

## Non-Negotiable Rule

Scientific visuals must be real source material from the main paper, SI, user-provided screenshots, or user-provided PPT pages. Do not create scientific-looking substitutes.

## Allowed Additions

You may add explanatory overlays to real figures:

- thin red boxes;
- arrows;
- light shaded regions;
- zoom windows copied from the same source figure;
- short Chinese labels;
- key numbers copied from the paper/SI;
- figure source labels such as `Fig. 2D` or `Fig. S11`;
- simple text-only logic arrows between text boxes when explaining the story.

## Forbidden Additions

Do not add:

- decorative vector icons;
- invented molecule/catalyst cartoons;
- invented crystal structures;
- generated microscopy images;
- generated spectra, curves, plots, or tables;
- fake mechanism schemes;
- AI-style science backgrounds;
- icons for instruments, atoms, reactors, cells, computers, or mechanisms unless those icons already exist in user-provided material.

## Annotation Discipline

Annotate only what helps oral explanation:

```text
key trend -> box or shaded region
key sample -> label or arrow
key number -> callout
key contrast -> side-by-side labels
```

Do not annotate every point. Too many arrows or labels make the page unreadable.

## Source Preservation

Preserve scientific meaning:

- keep axes, units, scale bars, legends, and sample names when needed;
- do not crop out conditions that explain the result;
- do not change colors or data encodings in a way that changes interpretation;
- do not smooth, replot, or redraw curves unless raw data are provided and the user explicitly requests a derived chart.
