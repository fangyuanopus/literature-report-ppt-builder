# Mainline and Backup Boundary

Use this reference when deciding what belongs in the main presentation story versus backup slides.

## Core Rule

The mainline deck should tell the paper's argument in a continuous sequence. Backup slides should support questions without interrupting that sequence.

## Mainline Pages Should Include

- paper identity and core contribution;
- field problem and unresolved gap;
- author's design strategy or method;
- key performance or function results;
- direct structural / mechanistic / validation evidence;
- only the most necessary controls;
- concise summary and inspiration.

## Backup Pages Should Include

- complete dense tables;
- detailed experimental conditions;
- full EXAFS fitting tables, raw parameters, or long method details;
- extra SI figures that support but do not drive the main story;
- full source panels before cropping;
- likely teacher/advisor questions and safe answers;
- alternative explanations and limitations.

## Boundary Tests

Ask before placing a slide in the mainline:

1. Does this page move the main argument forward?
2. Can the conclusion be understood in 30-60 seconds of oral explanation?
3. Is this evidence necessary for the next page to make sense?
4. Would removing this page break the story?

If the answer is mostly no, move it to backup.

## Backup Styling

Backup slides must still follow the same visual style and navigation system. Their navigation highlight should be `Backup`, and their titles should begin with a clear backup label such as:

```text
Backup 1: 合成条件与样品命名
Backup 2: 完整性能对比表
Backup 3: EXAFS 拟合参数
Backup 4: 可能问题与稳妥回答
```
